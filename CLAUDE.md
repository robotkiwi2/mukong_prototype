# 무협 타이핑 액션 게임

웹 기반 실시간 무협 액션 게임. 이동·방어·공격 등 모든 행위를 **타이핑(+엔터)으로 시전**한다.
구결을 외워 무공을 펼치는 핵심 판타지 — 타이핑 행위 자체가 세계관 내 무공 시전이다.

**상태:** 기획 단계 (다음 단계: 최소 전투 한 합 프로토타입)

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 장르 | 실시간 액션 / 무협 |
| 플랫폼 | 웹 (HTML/JS) |
| 입력 | 키보드 타이핑 + 엔터 |
| 기획문서 | `무협_타이핑_액션_게임_기획문서.md` |

---

## 핵심 설계 원칙

1. **입력이 곧 시전** — 타이핑과 무공 발현이 1:1 대응
2. **어구 길이 = 밸런싱 화폐** — 강한 무공일수록 길게 쳐야 함
3. **광역기로 직렬 입력 한계 해결** — 다중 위협은 광역 무공으로
4. **정확도 등급제(graded)** — 오타도 감소된 효과로 발현, binary 실패 없음
5. **지식(무학) vs 실력(내공·숙련) 분리** — 두 축이 독립적으로 성장

---

## 무공 구조

- **기본행위:** 이동, 회피, 발차기, 찌르기, 베기, 장풍 등 최소 단위
- **초식 = 기본행위의 명명된 조합** (함수/매크로 개념)
  - 입력은 압축되지만 실행 시간은 구성 행위 합만큼 그대로
- **단축 진화:** mastery 상승 → `풀네임 → 약어 → 초성 → (제한적) 단일키`
- **창안:** 플레이어가 나만의 초식 정의 (합성 상한 있음, 위력 비례 최소 이름 길이)

---

## 밸런싱 핵심

```
위력 비례 (입력 길이·내력·시전 시간) 하한선
  → 그 위에서 mastery가 모든 비용을 깎음
    → 창안은 합성 상한 안에서 입력을 묶음
```

- 하한선은 전역 상수가 아니라 **무공 위력에 비례**
- **시전 시간이 최후의 제동 장치** — 강한 무공은 통달해도 한 박자 남음

---

## 공간 및 이동

- **연속 공간** (헥스·격자 아님). 격자는 턴제로 미끄러질 위험 + 좌표 타이핑 부담 → 실시간성 우선이라 배제
- **탐험 ≠ 전투 분리:** 지역 노드 이동(비실시간) + 조우 시 전투 아레나(실시간). 무한 맵 아님
- **전투 아레나:** 플레이어는 중앙 앵커, 위협이 사방에서 접근(방사형). 거리 = 비행 시간 = 칠 수 있는 구결 길이
- **이동 = 방향 + 이동법(경공) 지정 → 캐릭터가 알아서 이동.** 정밀 좌표 조작 없음. 이동은 드문 전술 행위, 평소엔 방어·반격 구결에 손을 씀

## 아레나 구조

- **아레나 = 강호의 실재 장소** (소림사 경내 입구, 취선루 2층 등). 추상적 전투 상자 아님
- **핵심 원칙: 장소 정체성 → 메커닉 파생.** 좁은 주루라서 광역기 봉인·단축어 전용, 정파 성지라서 중립 살해 페널티 — 모든 수치가 "이곳이 어떤 곳인가"에서 나옴
- **Place(정체성+물리, 재사용) / Scene(장면: 모드·인물·웨이브·목표) 분리.** 같은 장소를 다른 장면으로 재방문 = 서사 장치
- **장면은 전투만이 아님 — 담판·대치·전투 모드 스펙트럼, 매끄럽게 전이.** 대치 = 첫 공격 구결 전의 기싸움(타이핑 게임 고유). "전투 시작" 화면 없음
- **좌표 = 정적은 직교(x,y, 중심 원점), 위협 판정은 극좌표(런타임 파생).** 방이 사각형 → 배치는 직교. 위협의 각도(단일/포위)·거리(=비행시간=구결 길이)는 매 프레임 플레이어 기준으로 계산, 저장 안 함
- **4블록:** 크기·형태(approachTime=경고시간) / 장애물(projectile·movement·sight 차단) / 인물(enemy·ally·neutral) / 지형(이동·시야·aoeRange·내력 변조)
- **장소 상태 = base → variant → override.** 같은 취선루가 성업/폐허로 — 물리(구조·지형·조명)만 패치(손님 유무는 Scene actors). obstacles는 id 기준 패치(remove/modify/add). 시나리오는 시간정보(연호·계절·시각·날씨)도 정의하며 장소 상태에 반영 가능

## 적 AI

- **룰 기반 상태 머신:** `Idle → Approach → WindUp → Attack → Cooldown` (+ Retreat/Defend 분기)
- 적은 타이핑 안 함 — 정해진 타이밍에 행동하는 행위자. 긴장감은 적 지능이 아니라 **공격 타이밍·속도**에서
- 난이도는 AI 지능이 아니라 **패턴 복잡도**(장풍 속도, 동시 공격 수, 포위, WindUp)로 조절

## 스토리 · 캠페인

- **원칙:** 새 초식 습득 = 새 타이핑 어휘 = 스토리 진행. 서사는 무공 습득의 동기
- **무림 편력 3막:** 입문(튜토=서사) → 강호(탐험·수련, 지역별 무공 계열) → 일가(창안·전수)
- **캠페인 = 데이터, 엔진 = 코드.** 캠페인 하나 = 데이터 파일 하나(등장 무공·웨이브·스토리·해금). "캠페인 추가 = 데이터 추가". 이 포맷이 곧 모딩 API
- 단, 첫 프로토타입은 하드코딩으로 핵심 루프 검증 후 데이터 추출

## 멀티플레이 방향

- **실시간 협동(Co-op) vs AI = 1순위.** "두 사람 = 두 문장"으로 직렬 입력 한계를 서사로 해결. 적이 AI라 지연에 관대 (호스트 권위 + WebRTC/WebSocket)
- **실시간 1:1 대전 = 포기** (지연 = 불공정)
- **비동기 PvP = 보류** (실시간성이 핵심이라 상충)

## 기술 고려사항

### 한글 입력 처리 (최우선 리스크)
- 조합형 IME 특성상 실시간 정오 판정이 까다로움
- `compositionend` 이벤트로 글자 완성 시점 캡처
- 자모 단위 vs 완성 글자 단위 판정 방식을 **가장 먼저 검증**할 것

### 입력 파서
- 기본행위(짧은 토큰) vs 초식(엔터로 확정되는 완성어) 문법적 분리
- 창안 문구는 네임스페이스 + longest-match로 충돌 해소

### 반응 시간 기준
- 숙련 타이피스트 ≈ 초당 5~7타 + 반응/판단 0.5초
- 비행 시간 1.5초 공격 대응 가능 어구 ≈ **3~5글자가 한계**
- 결론: 방어 단축어 ≤ 5글자, 긴 정식 초식명은 선제용

## 미해결 결정 사항

1. **정확도 판정 시점** — commit(완성 후) vs 스트리밍(실시간)
2. **시전 중 플레이어 행위** — 지켜보기 / 끊기·전환 / 다음 입력 버퍼링 중 선택
3. **하한선·시전 시간 구체 수치** — 프로토타입 플레이테스트로 결정
4. **타깃 유저** — 타이핑 애호가 vs 무협 팬 (난이도 게이트 강도 좌우)
